# NewsCollector SKILL

## 개요
Bloomberg, WSJ, Barron's의 기사 목록은 RSS CSV로 수집하고 개별 기사 본문은
- 기본: Chrome DevTools + 전용 Chrome 프로필
- 폴백: Chrome visible, Firefox 일반 모드 visible
경로로 확보하며, 월드 메모리를 참조해 중요 기사를 선별한 뒤 Axios 스타일 한국어 요약 파일을 생성하는 스킬.

---

## 트리거 명령어 ← 반드시 등록할 것

이 SKILL은 사용자의 요청문에 `Axios`, `악시오스`, `Axios식`, `악시오스식` 중 하나가 포함된 경우에만 실행한다.

아래와 같은 표현이 감지되면 이 SKILL을 즉시 실행한다.

- `Axios식 뉴스기사 요약 정리 기능 작동해줘`
- `악시오스식 뉴스 기사 정리해줘`
- `Axios 뉴스 수집해줘` / `악시오스 뉴스 정리해줘`
- `Axios 기사 업데이트해줘`
- `Axios식으로 WSJ 기사 정리해줘` / `악시오스식 Bloomberg 기사 정리해줘` / `Axios식 Barron's 기사 정리해줘`

아래처럼 `Axios/악시오스` 계열 표현이 없는 일반 요청은 이 SKILL을 발동하지 않는다.

- `뉴스 수집해줘`
- `뉴스 정리해줘`
- `WSJ 기사 정리해줘` / `Bloomberg 기사 정리해줘` / `Barron's 기사 정리해줘`
- 이런 요청은 기본적으로 `SKILLs/NewsRequest.md` 또는 일반 뉴스/시장 분석 흐름으로 처리한다.

## 사용 도구

- `scripts/safari_fetch.py` — RSS 기반 기사 목록 추출 + Chrome DevTools 메인 본문 수집 (`tmp/chrome_devtools_profile` 사용, 필요 시 `NEWS_FETCH_DEVTOOLS_PROFILE_PATH`로 다른 비표준 프로필 지정)
- `scripts/chrome_visible_fetch.py` — Chrome visible 보조 경로
- `scripts/firefox_visible_fetch.py` — Firefox 일반 모드를 보이는 창으로 직접 열어 기사 본문을 확보하고 탭을 닫은 뒤 앱까지 종료하는 최종 폴백 경로
- `scripts/news_update_harness.py` — 기사 본문 수집 하네스 + manifest 검증/반영
- `scripts/news_update_queue.py` — `.state.json` 기준 신규 기사 후보 확인
- `scripts/world_memory_cli.py` — 월드 메모리 컨텍스트 조회
- `/NewsUpdate/.state.json` — 소스별 마지막 수집 기사 URL 추적
- Bloomberg RSS CSV: `https://rss.app/feeds/_t07deORnyZW90CjC.csv`
- Dow Jones RSS CSV: `https://rss.app/feeds/_m6HwVpkVbkV6H1V6.csv`

## 실행 전 사전 준비 (매 실행 시)

1. 월드 메모리 최근 21일 조회:
   ```
   python3 scripts/world_memory_cli.py list --days 21 --format md
   ```
   → 현재 진행 중인 주요 이슈를 파악해 기사 중요도 판단에 활용한다.

2. `.state.json` 읽기 — 각 소스의 마지막 수집 URL 확인.
   파일이 없으면 `{}` 로 초기화.

3. 필요하면 신규 기사 후보 큐를 먼저 확인:
   ```
   python3 scripts/news_update_queue.py --workspace . --format md
   ```
   → 점수화 없이 `.state.json` 경계 기준으로 Bloomberg → WSJ → Barron's 신규 후보만 잘라 본다.

## 소스별 수집 절차

### 공통 원칙
- 헤드라인 목록 페이지에서 링크 목록을 추출한다.
- `.state.json`에 저장된 마지막 URL을 목록에서 찾는다.
  - 찾은 경우: 그 위(더 최신)의 기사만 대상으로 한다.
  - 찾지 못한 경우: 소스별 폴백 정책을 따른다 (아래 참조).
- 대상 기사 중 **월드 메모리 + 헤드라인 텍스트** 기반으로 중요도를 판단하고 **선별**한다.
- 선별된 기사만 개별 URL을 방문해 본문을 수집한다.
- 메인 본문 수집은 `chrome` 기본 경로(Chrome DevTools)를 우선 사용한다.
- visible 경로에서는 원격 디버깅, Playwright, WebDriver, BiDi를 사용하지 않는다.
- `chrome-visible`, `firefox-visible`은 보이는 일반 모드로만 다루고, 기사 본문을 확보하면 해당 탭을 닫고 앱도 종료한다.
- Bloomberg 접근은 기본 20초, Dow Jones 접근은 기본 10초 간격을 유지한다.
- `last_run_kst` 이후 신규 기사량이 **없거나 너무 적으면** `업데이트 할 기사 없음`으로 종료할 수 있다.
  - 예: 주말, 미국 심야시간, 장 마감 후 한산한 시간대
  - 이 경우 기사 파일이나 별도 보고서를 만들지 않는다.
  - 단, 단순히 적어 보인다는 이유만으로 건너뛰지 말고, 실제 신규 기사 수와 선별 가능한 기사 수를 확인한 뒤 결정한다.
- 개별 기사 본문 수집은 가능하면 `python3 scripts/news_update_harness.py fetch-article --url <article_url>` 또는
  `python3 scripts/news_update_harness.py fetch-batch --url <url1> --url <url2> ...`로 실행한다.
  - 기본 브라우저는 `chrome`이다.
  - `fetch-batch`는 `chrome` 기본 경로뿐 아니라 명시적으로 `--browser chrome-visible` 또는 `--browser firefox-visible`을 사용한 경우에도 작업 시작 시 브라우저를 1회 열고, 기사 본문을 모두 확보한 뒤 manifest 작업으로 넘어가기 전에 1회만 종료한다.
  - `fetch-batch`는 기본적으로 자동 브라우저 폴백을 사용하지 않는다. 배치 전체 지연을 감수하고서라도 폴백이 필요할 때만 `--allow-chrome-fallback`을 명시한다.
  - visible 경로를 직접 지정해야 할 때는 `--browser chrome-visible` 또는 `--browser firefox-visible`을 사용한다.
- **금지**: 임시 Python 스크립트 안에서 `python3 scripts/safari_fetch.py ...`를 여러 번 반복 호출하는 방식.
  DevTools 포트와 전용 Chrome 프로필 접근이 겹치면 세션이 꼬일 수 있으므로, 하네스의 진단+복구 재시도를 사용한다.
- 필요하면 먼저 `python3 scripts/safari_fetch.py https://www.wsj.com --session-setup`처럼 DevTools Chrome 프로필에 로그인한 뒤 본문 수집을 시작한다.

### Bloomberg
- 목록 소스: `https://rss.app/feeds/_t07deORnyZW90CjC.csv`
- 링크 추출: `python3 scripts/safari_fetch.py <URL> --load-more --source bloomberg`

- **Paywall 처리**: 개별 기사 본문 수집 시 paywall이 감지되면 1회 새로고침 후 재시도.
  ```
  python3 scripts/safari_fetch.py <article_url> --reload
  ```
  새로고침 후에도 paywall이면 **건너뛰고 오류 보고서에 기록**.
  - 단, 일반 본문 수집 시작점은 하네스 `fetch-article`/`fetch-batch`를 우선 사용한다.
- **폴백**: RSS 목록에서 마지막 URL을 찾지 못하면
  현재 RSS 창에 남아 있는 **가장 오래된 기사(맨 아래)**부터 전부 처리한다.

### Dow Jones (WSJ + Barron's)
- 목록 소스: `https://rss.app/feeds/_m6HwVpkVbkV6H1V6.csv`
- 링크 추출: `python3 scripts/safari_fetch.py <URL> --links-only --source dow_jones`
  - 구현상 `--links-only --source dow_jones`는 입력 URL 대신 Dow Jones RSS CSV를 사용한다.
  - 반환 JSON에는 각 기사별 `source=wsj|barrons`가 포함된다.
- 처리 방식:
  - Dow Jones 피드는 한 번만 읽고, 결과를 `WSJ`와 `Barron's`로 내부 분기한다.
  - `.state.json` 비교와 기사 선별은 `source`별로 따로 유지한다.
- **폴백**: RSS 목록에서 마지막 URL을 찾지 못한 경우,
  현재 RSS 창에 남아 있는 **가장 오래된 기사(맨 아래)**부터 전부 처리한다.

## 기사 중요도 판단 기준

아래 요소를 종합해 AI가 **상·중·하** 중요도를 판단한다.

0. **기사 형식은 자동 제외 사유가 아니다**: 오피니언, 칼럼, 사설, 분석 기사라도 시장·정책·산업·기업 영향이 크면 일반 기사와 동일하게 후보로 평가한다.
1. **월드 메모리 연관성**: 현재 진행 중인 이슈(지정학, 금리, 실적, 섹터 동향)와 관련도
2. **헤드라인 임팩트**: 시장·정책·기업에 직접 영향을 주는 내용 여부
3. **수집 시점 기준 신선도**: 동일 주제 기사가 이미 처리됐다면 중복 제외

### 사용자 취향 가중치 레이어

기본 중요도 판단 뒤에 아래 가중치를 한 번 더 적용한다.

1. **세계 각국의 경제·산업 통계 기사**는 기본적으로 **필수 포함 후보**로 승격한다.
   - 예: GDP, CPI, PPI, 고용, 생산, 수출입, 산업생산, 소매판매, PMI, 재고, 물동량, 관광·주택·소비 통계 등
2. **기업 실적 기사(earnings)** 는 기본적으로 **필수 포함 후보**로 승격한다.
   - 예: 실적 발표, 가이던스 상향·하향, beat/miss, 매출·이익 서프라이즈, 컨퍼런스콜 핵심 발언
3. **테크/AI 기사**도 기본적으로 **필수 포함 후보**로 승격한다.
   - 예: AI 모델 출시·평가, 반도체 수요·공급, 빅테크의 AI 전략, 클라우드·데이터센터 투자, 로보틱스, 사이버보안, AI 인프라 CAPEX
4. **한중일 3국 관련 기사**도 기본적으로 **필수 포함 후보**로 승격한다.
   - 예: 한국·중국·일본의 외교, 통상, 공급망, 산업정책, 안보, 공동성명, 정상회담, 수출규제, 기술협력, 갈등 조정
5. 위 네 유형은 일반 중요도가 `중` 이하로 보여도, **중복이 아니고 본문 수집이 가능하면 우선 포함**한다.
6. 동일 실행에서 필수 포함 후보가 여러 건이면, 중복 주제·중복 기업을 줄이면서도 최소 1건 이상 반영하는 쪽으로 우선순위를 조정한다.

**선별 규칙**: 중요도 **상** 기사는 모두 처리. **중** 기사는 소스당 최대 3건.
**하** 기사는 월드 메모리와 강한 연관이 없으면 제외.
- 오피니언/칼럼을 건너뛸 때도 이유는 `오피니언이어서`가 아니라 **시장·정책·산업 연관성이 약하거나 중복도가 높아서**여야 한다.

### 업데이트 없음 처리

아래 조건 중 하나에 해당하면 `업데이트 할 기사 없음`을 선택할 수 있다.

1. `.state.json`의 마지막 처리 시점 이후 세 소스를 합쳐도 **신규 기사 자체가 없는 경우**
2. 신규 기사는 있으나, 중복 제거와 중요도 판단 뒤 **실제로 남는 기사 수가 너무 적은 경우**
3. 남은 기사가 있어도 모두 잡음성 헤드라인이어서, 취향 가중치 레이어를 적용한 뒤에도 **기사화할 가치가 낮은 경우**

이 경우:

- 기사 `.md` 파일은 생성하지 않는다.
- 오류 보고서도 만들지 않는다. 단, 실제 오류가 발생한 경우는 예외다.
- 실행 결과는 간단히 `업데이트 할 기사 없음`으로만 처리한다.

## 출력 파일 규칙

- **저장 경로**: `/NewsUpdate/`
- **파일명**: `YY-MM-DD HH-MM {한국어 제목}.md`
  - 날짜·시간 기준: **수집 시각 (KST)**
  - 예시: `26-04-09 14-32 연준, 금리 동결 결정.md`
- **구조**: 기사 1건 = 파일 1개
- **인코딩**: UTF-8

## 오류 처리 및 오류 보고서

다음 상황에서 `/NewsUpdate/ERROR-YY-MM-DD HH-MM.md` 파일을 생성한다.

| 상황 | 처리 |
|------|------|
| 로그인 풀림 (비로그인 상태 감지) | 오류 보고서 작성 후 해당 소스 건너뜀 |
| Bloomberg paywall 새로고침 후에도 지속 | 해당 기사 URL 오류 보고서에 기록 후 건너뜀 |
| Chrome DevTools 본문 수집 실패 | 오류 보고서 작성 |
| 기타 예외 | 오류 보고서 작성 |

**로그인 이탈 감지 신호**: 페이지 텍스트에 "sign in", "log in", "create account", "subscribe" 등이 포함되고 본문이 400자 미만인 경우.

오류 보고서 형식:
```
# 오류 보고서
- 발생 시각 (KST): ...
- 소스: Bloomberg / WSJ / Barron's
- 대상 URL: ...
- 오류 유형: 로그인 이탈 / Paywall / DevTools 오류 / 기타
- 상세 메시지: ...
```

## 상태 파일 관리

**경로**: `/NewsUpdate/.state.json`

**형식**:
```json
{
  "wsj": "https://www.wsj.com/articles/last-read-article",
  "barrons": "https://www.barrons.com/articles/last-read-article",
  "bloomberg": "https://www.bloomberg.com/news/articles/last-read-article",
  "last_run_kst": "2026-04-09T14:32:00+09:00"
}
```

- 각 소스 처리가 **완료될 때마다** 해당 소스의 최신 처리 URL을 즉시 업데이트한다.
- 전체 실행 완료 후 `last_run_kst`를 갱신한다.

### 출력 커밋 하네스

- 기사와 오류 보고서, 그리고 갱신할 `.state.json`은 **직접 `/NewsUpdate/`에 먼저 쓰지 말고** manifest JSON으로 묶는다.
- manifest 예시 구조:
```json
{
  "articles": [
    {
      "filename": "26-04-10 20-08 예시 기사.md",
      "content": "# ..."
    }
  ],
  "errors": [
    {
      "filename": "ERROR-26-04-10 20-08.md",
      "content": "# 오류 보고서\n..."
    }
  ],
  "state": {
    "last_run_kst": "2026-04-10T20:08:00+09:00",
    "bloomberg": "https://...",
    "wsj": "https://...",
    "barrons": "https://..."
  }
}
```
- 반영 전에 아래 명령으로 검증한다.
  `python3 scripts/news_update_harness.py validate-manifest --manifest /tmp/news_batch.json`
- 검증이 통과한 경우에만 아래 명령으로 최종 반영한다.
  `python3 scripts/news_update_harness.py apply-manifest --manifest /tmp/news_batch.json --workspace .`
- 검증이 실패하면 `.state.json`을 갱신하지 않는다.
- 하네스는 기사 길이, Axios 전환구 형식, 리스트 밀도, 출처 링크, 금융 에이전트 표기, state 구조를 함께 검사한다.

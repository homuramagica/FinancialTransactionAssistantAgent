# 금융 거래 어시스트 사용자 도움말 (주붕이 에디션)

안녕하세요, **아카라이브 주식채널 주붕이 여러분**.  
저는 **금융 거래 어시스트 에이전트**입니다.

이 문서는 이 프로젝트의 인스트럭션 문서들을 바탕으로, 기술에 익숙하지 않은 일반 투자자도 바로 쓸 수 있게 정리한 실전 가이드입니다.  
목표는 하나입니다.

**"복잡한 건 제가 맡고, 주붕이님은 좋은 질문과 좋은 판단만 하자."**

## 0) 10초 요약

- 기본 데이터 소스는 `yfinance`입니다.
- 뉴스/속보 요청이면 FEED를 모아서 최신 정보를 보여줍니다.
- 기업/섹터/포트폴리오 분석은 기본적으로 `yfinance + 고신뢰 웹 검색` 중심입니다.
- 포트폴리오 로그와 월드 메모리 로그를 쌓으면, 상담 품질이 점점 좋아집니다.
- 차트는 범례 대신 오른쪽 끝 라벨로 보기 쉽게 만들어집니다.

## 2) 시작 전 준비 (아주 쉽게)

### 2-1. 설치 확인

```bash
python3 --version
python3 -c "import yfinance; print(yfinance.__version__)"
```

두 번째 명령에서 에러가 나면 아직 `yfinance`가 없는 상태입니다.

### 2-2. 권장 설치

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

또는 `uv`를 쓰면:

```bash
uv pip install --python .venv/bin/python -r requirements.txt
```

팁:
- 가상환경을 안 켠 상태라면 `python3` 대신 `.venv/bin/python`으로 실행하면 됩니다.
- 설치가 필요한 상황이면, 원칙상 먼저 설치 진행 여부를 물어보고 진행합니다.

### 2-3. 터미널이 두려우신 주붕이님용 초간편 가이드

터미널이 두려우신 주붕이님들은, 에이전트에게 **권한을 충분히 올려주고** 직접 설치하라고 요청해 보세요.  
저는 금융 에이전트로서 설치 확인부터 환경 점검, 기본 패키지 설치까지 순서대로 도와드릴 수 있습니다.

복붙 요청 예시:

```text
터미널이 익숙하지 않아서요. 설치 권한 충분히 드릴 테니 이 프로젝트 실행에 필요한 패키지를 직접 점검하고 설치해줘.
```

```text
yfinance 포함해서 필요한 패키지 설치부터 실행 확인까지 한 번에 진행해줘. 중간에 뭐 하는지 쉽게 설명도 같이 부탁해.
```

작은 팁:
- "무엇을 왜 설치하는지"를 함께 설명해 달라고 하면 훨씬 덜 불안합니다.
- 설치 후에는 "실행 테스트까지 해줘"라고 한 줄 더 붙이면 품질이 올라갑니다.

## 3) 질문만 잘 던져도 되는 "대화형 기능" 가이드

아래 문장을 그대로 복붙해도 됩니다.

### 3-1. 뉴스/속보

언제 좋나:
- 장중 급변, 새벽 속보, "방금 나온 뉴스"가 궁금할 때

복붙 질문:
- `최신 글로벌 금융 속보 정리해줘. KST 기준 시간으로 보여줘.`
- `방금 업데이트된 뉴스만 다시 알려줘.`

특징:
- 텔레그램 FEED + 정규 언론 FEED + 원본 텔레그램 채널을 함께 확인합니다.
- 오피니언 기사는 별도 섹션으로 분리합니다.

### 3-2. 시장 상황 분석

언제 좋나:
- "오늘 왜 리스크오프냐?" 같은 장 해석이 필요할 때

복붙 질문:
- `지금 시장 리스크 온/오프 판단해줘. Flash + Deep-Dive 둘 다.`
- `오늘 하락 원인을 주식/변동성/금리/달러 관점으로 정리해줘.`

특징:
- `결론` 섹션으로 마무리합니다.
- 단정 대신 조건과 트리거 중심으로 제시합니다.

### 3-3. 거시경제 보고서

언제 좋나:
- 국가/지역 전망이 궁금할 때

복붙 질문:
- `미국 거시경제 6~12개월 전망을 Baseline/Upside/Downside로 정리해줘.`
- `한국 성장률/물가/정책 경로 보고서 형식으로 작성해줘.`

특징:
- 고신뢰 매체/기관(WSJ, FT, Bloomberg, Reuters, IMF 등) 중심.
- 지표 나열보다 동인(왜 그런지)에 초점을 둡니다.

### 3-4. 기업 실적(earnings)

언제 좋나:
- 실적 발표 전후 체크가 필요할 때

복붙 질문:
- `AAPL 최근 4분기 실적과 컨센서스 변화 요약해줘.`
- `NVDA 실적 발표 전 체크리스트 만들어줘.`

특징:
- yfinance 중심으로 실적일/서프라이즈/추정치/재무를 묶습니다.
- 필요하면 공시(SEC/DART)까지 보강합니다.

### 3-5. 기업 분석 (투자리포트)

언제 좋나:
- 기업을 본격적으로 평가하고 싶을 때

복붙 질문:
- `MSFT 기업 분석 보고서 써줘. Flash + 심층으로.`
- `코카콜라 vs 펩시 피어 비교해줘.`
- `초보자용으로 이 회사가 돈 버는 구조만 쉽게 설명해줘.`

특징:
- 기본은 `yfinance + 고신뢰 웹 검색`.
- 속보를 명시적으로 원할 때만 FEED를 추가합니다.

### 3-6. 섹터 분석

언제 좋나:
- 어느 섹터가 유리한지, 로테이션이 도는지 볼 때

복붙 질문:
- `미국 섹터 중 최근 3개월 강한 순서와 이유 알려줘.`
- `기술 vs 에너지 섹터 비교 리포트 써줘.`

특징:
- 정량만 보지 않고 뉴스/정책/수급/내러티브를 반드시 함께 봅니다.
- 기본 소스는 `yfinance + 고신뢰 웹 검색`.

### 3-7. 포트폴리오 상담

언제 좋나:
- "내 비중 이대로 괜찮나?"가 걱정될 때

복붙 질문:
- `내 포트폴리오 리밸런싱 제안해줘.`
- `급락장 방어 플랜 만들어줘.`
- `은퇴까지 20년 기준 라이프사이클 계획 짜줘.`

특징:
- 행동편향, 목표, 투자기간까지 반영합니다.
- 필요 시 월드 메모리 로그를 함께 참고합니다.

### 3-8. 초보자 모드

언제 좋나:
- 전문 용어가 버거울 때

복붙 질문:
- `초보자 모드로 설명해줘.`
- `어려워요. 한 단계씩 다시 설명해줘.`

특징:
- 짧은 문장, 작은 개념 단위, 쉬운 예시 중심으로 설명합니다.

## 4) 분석 결과, 이렇게 읽으면 훨씬 쉽습니다

주붕이님이 리포트를 받았을 때, 숫자가 많아도 겁먹을 필요 없습니다.  
아래 4단계만 보면 됩니다.

### 4-1. 1단계: 한 줄 결론부터 본다

- 먼저 `한 줄 결론`만 읽으세요.
- 여기서 "지금은 공격/중립/방어 중 어디인지"를 파악합니다.
- 결론이 길면, 아직 결론이 아닌 경우가 많습니다.

### 4-2. 2단계: 근거 3개만 고른다

- 모든 근거를 다 볼 필요 없습니다.
- 딱 3개만 고르세요.
- 예시: `금리`, `변동성`, `실적 추정치 변화`

요령:
- 서로 다른 성격의 근거를 고르면 좋습니다.
- 같은 이야기 3번 반복한 근거는 1개로 취급하세요.

### 4-3. 3단계: 내 포트에 연결한다

아무리 좋은 분석도 "내 비중"에 연결 안 되면 구경으로 끝납니다.

연결 질문:
- 이 결론이 내 포트에서 가장 아픈 구간은 어디인가?
- 비중을 지금 당장 바꿔야 하나, 관찰만 하면 되나?
- 손실이 커질 때 중단할 기준은 무엇인가?

### 4-4. 4단계: 액션을 "작게" 실행한다

- 한 번에 대수술 하지 마세요.
- 1차 조정 후 반응을 보고 2차 조정으로 나누세요.
- 시장은 드라마, 계좌는 다큐로 운영하는 게 보통 유리합니다.

## 5) 시장/기업/섹터 분석에서 자주 나오는 용어, 초간단 번역

| 용어 | 쉬운 뜻 | 주붕이식 해석 |
| --- | --- | --- |
| 리스크 온(Risk-on) | 위험자산 선호 | "사람들이 공격적으로 간다" |
| 리스크 오프(Risk-off) | 안전자산 선호 | "사람들이 방패부터 든다" |
| 컨센서스(Consensus) | 시장 평균 기대치 | "다들 이 정도는 예상했다" |
| 서프라이즈(Surprise) | 기대 대비 실제 차이 | "예상보다 좋냐/나쁘냐" |
| 밸류에이션(Valuation) | 가격이 비싼지 싼지 | "가격표가 내용물 대비 어떤지" |
| 드로우다운(Drawdown) | 고점 대비 하락폭 | "멘탈 흔들리는 구간 수치화" |
| 변동성(Volatility) | 가격 흔들림 강도 | "심장박동 그래프의 진폭" |
| 트리거(Trigger) | 방향 바꿀 계기 | "판 뒤집는 조건" |

짧은 팁:
- 모르는 용어가 나오면 "이 용어 초보자식으로 1문장만" 요청하면 됩니다.

## 6) 월드 메모리 기능, 왜 진짜 유용한가?

짧게 말하면:
- 속보는 "지금 화재 경보기"
- 월드 메모리는 "화재 원인과 동네 소방지도"

즉, 단발 뉴스 요약이 아니라 **지속 이슈의 누적 기억**입니다.

필수 저장 항목:
- `as_of` (KST)
- `category`: `stock_bond | geopolitics | emerging`
- `region`: `US | KR | GLOBAL`
- `importance`: `high | medium | low`
- `sources` (매체/URL/게시시각)

활용법:
1. 상담 전에 최근 14~30일 로그를 조회
2. 미국/한국/글로벌 + 정치/신흥이슈를 분리
3. 현재 포트폴리오 액션(비중 조정, 방어 강화, 관찰 지표)으로 연결

## 7) 오토메이션 추천 (하루 1~2회)

주붕이님이 자동화를 켤 수 있는 환경이라면, 저는 이렇게 권장합니다.

### 7-1. 자동화 프롬프트 예시

아래 문장을 자동화 작업 프롬프트로 사용:

`월드 메모리 확보 작업을 수행하세요.`

조금 더 구체화한 추천 프롬프트:

`월드 메모리 확보 작업을 수행하세요. 최근 시장/정치/신흥 이슈를 고신뢰 출처 기준으로 업데이트하고, portfolio/world_issue_log.jsonl에 누락 항목을 보강한 뒤, reports/ 경로에 일일 요약 보고서를 생성하세요.`

### 7-2. 추천 실행 빈도

- 보수형: 하루 1회 (예: KST 오전 8시 30분)
- 적극형: 하루 2회 (예: KST 오전 8시 30분 + 오후 8시 30분)

### 7-3. 자동화가 하면 좋은 일

1. 최근 로그(예: 14일) 조회
2. 누락된 중기 이슈 보강
3. 출처 필수 필드 점검
4. `world_memory_report_YYYY-MM-DD.md` 생성
5. 다음 상담에서 바로 참조 가능하게 정리

한 줄 유머:
- 이걸 안 켜면 주붕이의 뇌는 SSD, 켜면 NAS가 됩니다.

## 8) 자주 하는 실수 (미리 방지)

1. `속보`와 `중기 동향`을 섞어버리기  
   - 해결: 속보는 뉴스 요청, 흐름은 월드 메모리 요청으로 분리

2. 차트만 보고 결론 내리기  
   - 해결: 변동성/금리/달러/크레딧을 최소 2개 이상 교차확인

3. "느낌상" 리밸런싱  
   - 해결: 현재 비중, 최근 성과 흐름, 월드 메모리 요약을 먼저 확인

4. 모든 걸 한 번에 바꾸기  
   - 해결: 단계적 리밸런싱(1차/2차)으로 나눠 실행
